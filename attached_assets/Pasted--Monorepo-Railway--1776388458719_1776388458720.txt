🚀 **توجيه: نشر مشروع Monorepo كامل على Railway (جاهز للإنتاج، بدون أي أخطاء)**

يجب عليك نشر هذا المشروع بالكامل على Railway بشكل صحيح وكامل.

---

# 🎯 الهدف

خذ المشروع الحالي الموجود على Replit وقم بـ:

* رفعه إلى GitHub
* ضبطه بشكل صحيح للعمل على Railway
* نشر جميع الخدمات بنجاح
* التأكد أن التطبيق يعمل أونلاين بدون أي أخطاء

---

# 📦 نوع المشروع

هذا المشروع هو **Monorepo باستخدام pnpm** ويحتوي على هيكل مشابه:

* apps/api-server
* apps/poultry-manager
* apps/mockup-sandbox

❗ يجب التعامل معه كـ Monorepo (وليس تطبيق واحد)

---

# 🚀 خطوات النشر (إلزامية)

## 1. تجهيز المشروع

تأكد من:

* وجود ملف pnpm-workspace.yaml
* جميع الحزم (dependencies) صحيحة
* عدم وجود أخطاء في الاستيراد (imports)
* المشروع يعمل ويتم بناؤه (build) محليًا بدون مشاكل

---

## 2. الرفع إلى GitHub

* تهيئة git إذا لم يكن موجود
* حفظ جميع الملفات (commit)
* رفع المشروع إلى مستودع GitHub

---

## 3. إعداد Railway

إنشاء **خدمات منفصلة لكل جزء من التطبيق**:

---

### 🥇 الخدمة 1: api-server

* Root Directory:
  `apps/api-server`

* Build Command:
  pnpm install --no-frozen-lockfile && pnpm build

* Start Command:
  pnpm start

---

### 🥈 الخدمة 2: poultry-manager

* Root Directory:
  `apps/poultry-manager`

* Build Command:
  pnpm install --no-frozen-lockfile && pnpm build

* Start Command:
  pnpm start

---

### 🥉 الخدمة 3: mockup-sandbox (الواجهة)

* Root Directory:
  `apps/mockup-sandbox`

* Build Command:
  pnpm install --no-frozen-lockfile && pnpm build

* Start Command:
  pnpm start

---

## 4. إعداد قاعدة البيانات

* إنشاء قاعدة بيانات PostgreSQL داخل Railway
* ربطها باستخدام DATABASE_URL
* التأكد أن الـ backend يتصل بها بدون أخطاء

---

## 5. متغيرات البيئة (Environment Variables)

* إضافة جميع المتغيرات المطلوبة
* التأكد من عدم وجود أي إعداد ناقص

---

## 6. إعداد البورت

يجب التأكد أن السيرفر يستخدم:

process.env.PORT

---

## 7. التحقق من النشر

يجب التأكد من:

* نجاح عملية البناء (Build)
* تشغيل جميع الخدمات بشكل صحيح
* عدم وجود أخطاء أثناء التشغيل
* الواجهة (Frontend) متصلة بالـ Backend
* قاعدة البيانات تعمل بشكل سليم
