PROJECT: FarmX AI – Production Hardening & Deployment Readiness

This system must be treated as a production-grade platform. All items below are mandatory before deployment.

---

## 🔴 SECTION 1 — CRITICAL SECURITY (P0)

1. REMOVE DEFAULT PASSWORDS

* Delete any seed logic that creates users with static passwords (e.g. 1234).
* If seeding is required:

  * Use environment variables (ADMIN_EMAIL, ADMIN_PASSWORD).
  * Or implement a one-time setup route (disabled after first run).
* Add runtime check:
  IF default credentials detected → TERMINATE app.

2. SESSION SECURITY

* Remove fallback:
  ❌ "farm-secret-key-change-me"
* Enforce:
  process.env.SESSION_SECRET must exist.
* If missing in production → throw error and exit.
* Use strong session config:

  * httpOnly: true
  * secure: true (in production)
  * sameSite: "lax" or "strict"

3. PASSWORD HANDLING

* Ensure bcrypt is used with saltRounds ≥ 10
* Never log passwords or hashes
* Validate password strength (min length ≥ 8)

---

## 🟠 SECTION 2 — AUTHORIZATION (RBAC)

4. IMPLEMENT ROLE-BASED ACCESS CONTROL

* Roles:

  * admin
  * worker

* Create middleware:
  requireAuth()
  requireRole("admin")

* Apply rules:

  * GET → allowed for authenticated users
  * POST/PUT/DELETE → restricted:

    * admin: full access
    * worker: limited (define scope)

* DO NOT rely on frontend checks

---

## 🟠 SECTION 3 — API HARDENING

5. INPUT VALIDATION

* Ensure all routes use Zod validation
* Reject invalid schemas (400)

6. ERROR HANDLING

* No stack traces in production
* Return structured error:
  { success: false, message: "..." }

7. RATE LIMITING

* Apply to:

  * login
  * auth routes
* Example: 5 attempts / minute
