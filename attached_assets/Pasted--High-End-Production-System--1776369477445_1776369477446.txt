🚨 طلب تطوير شامل للنظام – بناء نسخة احترافية متقدمة (High-End Production System)

أريد منك التعامل مع هذا المشروع كنظام احترافي كامل وليس تطبيق بسيط.  
المطلوب هو إعادة بناء المنطق الداخلي (Backend + Database + System Logic) ليصبح النظام:

- دقيق 100% في البيانات
- قابل للتوسع (Scalable)
- بدون أي تضارب بيانات (Zero Data Conflict)
- جاهز مستقبلاً للذكاء الاصطناعي والتحليل المتقدم

❗ لا أريد حلول مؤقتة أو ترقيع — أريد بناء هندسي صحيح من الأساس.

---

# 🧠 المطلوب الأساسي

تحويل النظام إلى Architecture احترافي قائم على فصل الأنظمة (Domain-Driven Design)

---

# 🧩 1. فصل الأنظمة بالكامل (Domain Separation)

يجب فصل كل جزء كنظام مستقل:

- Egg System (البيض)
- Feed System (العلف)
- Expense System (المصاريف)
- Incubation System (التفقيس)
- Activity System (النشاط)
- Task System (المهام)

🚫 ممنوع خلط أي بيانات بين هذه الأنظمة

---

# 🏗️ 2. إعادة بناء قاعدة البيانات (Clean Architecture)

❌ ممنوع استخدام جدول واحد يحتوي كل العمليات

✅ المطلوب:

إنشاء جداول مستقلة:

- egg_transactions
- feed_transactions
- expense_transactions

كل جدول يحتوي:

- id
- quantity أو amount
- cost
- date
- source
- notes

---

# 🔒 3. نظام Types صارم

🚫 ممنوع استخدام نصوص مثل "بيض" أو "علف"

✅ استخدم:

- Enum أو Constants ثابتة داخل الكود

---

# 🔗 4. بناء طبقة تحليل مستقلة (Service Layer)

إنشاء Service مركزي مثل:

FarmAnalyticsService

مسؤول عن:

- جمع البيانات من كل الأنظمة
- حساب التكاليف
- استخراج التقارير
- تحليل الأداء

🚫 ممنوع الحساب المباشر داخل الواجهة

---

# 🧠 5. Single Source of Truth

كل نوع بيانات له مصدر واحد فقط:

- البيض → egg_transactions
- العلف → feed_transactions

🚫 أي تكرار أو خلط = مرفوض

---

# ⚙️ 6. Validation Layer (إجباري)

قبل حفظ أي بيانات:

- التحقق من النوع
- التحقق من المكان الصحيح
- منع أي إدخال خاطئ

---

# 🚨 7. إصلاح الأخطاء الحالية

## مشكلة البيض:
- إنشاء تصنيف مستقل "بيض"
- أي عملية تخص البيض لا تدخل "أخرى"

## مشكلة العلف:
- منع إضافة البيض داخل العلف نهائياً
- فصل الحسابات بشكل كامل

---

# 🚀 8. تطوير نظام التفقيس (Real-Time Live System)

تحويله إلى نظام مباشر (Live Tracking):

- عرض اليوم الحالي
- عرض الساعة
- نسبة التقدم %
- الوقت المتبقي

المعادلة:

Progress = (Current Time - Start Time) / Total Time × 100

---

التنفيذ:

- WebSocket أو Firebase Realtime
- تحديث كل دقيقة

---

الواجهة:

- Progress Bar متحرك
- ألوان ديناميكية حسب المرحلة

---

# 🔄 9. دمج النشاط مع المهام (Smart Integration)

إنشاء قسم موحد:

📂 "مركز العمليات اليومية"

يحتوي:

- المهام اليومية
- سجل النشاط

---

الربط:

- أي Activity يتم تسجيله:
  - يتحول إلى Task
  - أو يرتبط بـ Task موجود

---

# 🧠 10. إضافة ذكاء داخلي (Smart System)

- عند إدخال أي نشاط:
  - يقترح تصنيف تلقائي
  - يقترح ربطه بمهمة

---

# 🎯 متطلبات إلزامية

- Clean Code
- Modular Design
- Scalable Architecture
- قابل للتطوير مستقبلاً بدون إعادة بناء

---

# ⚠️ تحذير مهم

أي خطأ في فصل البيانات الآن سيؤدي إلى:

- تقارير خاطئة
- قرارات خاطئة
- خسائر مالية

❗ لذلك أريد تنفيذ احترافي 100% وكأنك تبني نظام لشركة كبيرة

---

# 🧨 الهدف النهائي

أريد نظام:

- ذكي
- دقيق
- سريع
- قابل للتحليل والتطوير
- ويمكن لاحقاً ربطه بالذكاء الاصطناعي والتوقعات
